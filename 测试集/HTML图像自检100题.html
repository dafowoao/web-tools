🖼 HTML & 图像自检能力评测

10维度 × 10题 = 100题 · 每题描述一个常见HTML/图像陷阱

📐 D1 视口与响应式

响应式 D1-1: <meta name="viewport" content="width=device-width"> 缺少什么？
✅ 缺少 initial-scale=1.0，否则iOS Safari会缩放页面。
响应式 D1-2: width=2560 作为viewport有什么问题？
✅ 固定宽度viewport导致手机上看是整体缩小，字体小到不可读。应用 width=device-width。
CSS D1-3: 为什么 position: fixed 在iOS Safari上会抖动？
✅ iOS Safari的工具栏会动态伸缩，100vh 不等于可视高度。应用 dvh 或 svh。
响应式 D1-4: 只用 @media (max-width: 768px) 做移动端适配漏掉了什么？
✅ 漏掉了 1024px（平板横屏）、1440px（小屏笔记本），应从大到小至少4级断点。
响应式 D1-5: overflow-x: hidden 在body上有什么隐患？
✅ 可能导致绝对定位/fixed元素被意外裁剪，尤其是有弹窗时。应只在特定容器上用。
响应式 D1-6: 为什么 max-width: 100vw 在滚动条存在时仍会溢出？
✅ 100vw 包含滚动条宽度（约17px），实际可用宽度小于此值。应用 100%。
CSS D1-7: touch-action: manipulation 什么时候必须加？
✅ 移动端按钮/可交互元素上不加，iOS会有300ms点击延迟。
响应式 D1-8: rem 和 em 在响应式中的典型误用？
✅ 在 font-size 上用 em 会级联累积（嵌套深了越来越大），布局间距应用 rem。
响应式 D1-9: 图片在Grid中为什么有时比容器大？
✅ Grid子项默认 min-width: auto，图片会按原始尺寸撑大。应加 min-width: 0 或 overflow: hidden。
CSS D1-10: clamp() 函数在处理流体排版时的回退方案？
✅ 老浏览器不支持 clamp()，应写一个固定值作为fallback：font-size: 2rem; font-size: clamp(1.5rem, 3vw, 2.5rem);

⚡ D2 性能与加载

性能 D2-1: <script> 在 <head> 中不加 defer 有什么问题？
✅ 阻塞HTML解析直到脚本下载执行完，显著延迟首屏渲染。应加 defer 或放body末尾。
性能 D2-2: <link rel="stylesheet" href="..."> 在 <head> 中会阻塞什么？
✅ 阻塞首次渲染（FCP），浏览器必须等CSSOM构建完才渲染。关键CSS应内联，非关键用 media="print" onload="this.media='all'"。
性能 D2-3: 加载Google Fonts时最常见的性能问题？
✅ display=swap 不加会导致文字在字体加载期间不可见（FOIT），最长阻塞3秒。应始终加 &display=swap。
性能 D2-4: @import url(...) 和 <link> 哪个更好？
✅ @import 在CSS文件中间时导致瀑布式下载，比 <link> 至少多一次往返。永远用 <link>。
性能 D2-5: 为什么Font Awesome的JS版本比CSS版本慢？
✅ JS版本在DOM构建后扫描页面替换图标的，延迟图标显示且消耗CPU。CSS版本 (woff/woff2 + class) 更快。
性能 D2-6: 为什么 resize 事件监听器会导致卡顿？
✅ resize触发频率极高（每秒几十次），不加防抖/节流会导致大量布局计算。应用 requestAnimationFrame 或 debounce。
性能 D2-7: will-change: transform 滥用的后果？
✅ 告诉浏览器提前准备GPU层，但用得太多会耗尽GPU内存（移动端尤其）。只对正在动画的元素短期用。
性能 D2-8: 为什么 requestAnimationFrame 比 setInterval(fn, 16) 做动画更好？
✅ rAF在浏览器下次绘制前执行，自动匹配刷新率（60/120/144fps），标签页不可见时暂停。setInterval不管用户看不看都跑。
性能 D2-9: 埋点脚本中的 <script async> 和 defer 选哪个？
✅ 埋点用 async（下载完就执行，不保证顺序，不阻塞渲染）。defer 保证DOMContentLoaded前执行。
性能 D2-10: content-visibility: auto 什么时候用？
✅ 对离屏内容（长页面下方的卡片），延迟渲染直到接近视口。但不应用在首屏内容上，否则CLS会恶化。

♿ D3 可访问性（Accessibility）

无障碍 D3-1: 纯图标按钮没有文字标签有什么问题？
✅ 屏幕阅读器无法朗读。应加 aria-label 或 <span class="sr-only">。
无障碍 D3-2: 颜色对比度不够的最小标准？
✅ 正常文字 WCAG AA 要求 4.5:1，大文字(≥18px粗体/24px) 要求 3:1。用 DevTools 检查。
无障碍 D3-3: 只用颜色（红色/绿色）区分状态有什么问题？
✅ 红绿色盲用户（占男性8%）无法区分。应加图标/文字/形状辅助区分。
无障碍 D3-4: tabindex > 0 为什么是反模式？
✅ 破坏了自然的Tab键导航顺序。始终用 tabindex="0"（加入Tab顺序）或 tabindex="-1"（可编程聚焦但不在Tab顺序中）。
无障碍 D3-5: img标签缺少alt属性的后果？
✅ 屏幕阅读器会读文件名。装饰性图片用 alt=""（空alt），有内容的图片用描述性文本。
无障碍 D3-6: 自定义复选框（隐藏原生input）常见问题？
✅ 隐藏input后没把焦点行为移过来，键盘无法操作。应保留 opacity:0; position:absolute 让焦点仍在原input上。
无障碍 D3-7: aria-hidden="true" 在可聚焦元素上有什么后果？
✅ 屏幕阅读器隐藏元素但键盘仍可聚焦，导致"幽灵焦点"：聚焦在看不见的元素上。永远不要对可聚焦元素用 aria-hidden。
无障碍 D3-8: 弹窗打开后焦点没管理的后果？
✅ 焦点停留在背后的元素上，屏幕阅读器继续读背景内容。打开弹窗时应 focus() 到弹窗，关闭时回到触发元素。
无障碍 D3-9: 为什么 role="alert" 不能放在静态文本上？
✅ role="alert" 只对动态插入/变化的内容有效，静态文本不会触发朗读。应通过JS动态插入。
无障碍 D3-10: prefers-reduced-motion 媒体查询什么时候必须加？
✅ 用户系统设置了"减少动效"时，大动画/闪烁会引发眩晕。对 @keyframes 和 scroll-behavior: smooth 应用此查询。

🎨 D4 CSS陷阱

CSS D4-1: z-index: 9999 为什么是坏习惯？
✅ 数字大战最后谁也盖不住谁，应在父容器上建新的堆叠上下文（isolation: isolate），然后用 z-index: 1/2/3。
CSS D4-2: flex: 1 和 flex-grow: 1 的区别？
✅ flex: 1 是 flex-grow: 1; flex-shrink: 1; flex-basis: 0%，而 flex-grow: 1 保留原始 flex-basis: auto。大多数情况应用 flex: 1。
CSS D4-3: 为什么百分比高度在嵌套flex/grid中不生效？
✅ 百分比高度需要父容器有明确高度（非auto）。父容器没有设定 height 时，height: 100% 无效。
CSS D4-4: gap 在旧版flexbox浏览器中有什么问题？
✅ Safari 14.1以下不支持flex/ gap，需用 margin 做降级。2025年已基本不是问题。
CSS D4-5: outline: none 在按钮上不加 outline: 2px solid ... 的后果？
✅ 键盘导航用户看不到焦点位置。应始终保留焦点样式，至少 outline: 2px solid Highlight。
CSS D4-6: transform: translate(-50%, -50%) 居中为什么会模糊？
✅ 百分比 translate 在奇数次像素上时（如50.5px），文字会模糊。应用 calc 固定像素或确保容器宽高为偶数。
CSS D4-7: box-sizing: border-box 不加在所有元素上有什么隐患？
✅ 默认 content-box 时 width: 100%; padding: 20px 溢出父容器。应全局设 *, *::before, *::after { box-sizing: border-box }。
CSS D4-8: img { width: 100%; height: auto } 为什么有时不管用？
✅ 父容器没有宽度限制时图片不会缩小。且 max-width: 100% 比 width: 100% 更好——它允许小图不变大模糊。
CSS D4-9: visibility: hidden 和 display: none 在动画中的区别？
✅ display: none 触发重排（layout），不能做过渡动画。隐藏用 visibility（只触发重绘+合成），移除DOM用 display。
CSS D4-10: 为什么 font-display: block 在自定义字体上会伤害用户体验？
✅ block 导致最长3秒文字不可见（FOIT），之后才回退。应用 swap 立即显示回退字体，加载完后替换。

🏗 D5 HTML语义

语义 D5-1: 用 <div onclick="..."> 而不是 <button> 有什么问题？
✅ 不支持键盘操作（Enter/Space）、屏幕阅读器不识别为可交互元素、无默认焦点样式。
语义 D5-2: 为什么 <br> 不能用于段落间隔？
✅ <br> 只是换行，不增加段落间距。段落间隔应用 <p> 的 margin。
语义 D5-3: 页面没有 <h1> 或多个 <h1> 的后果？
✅ SEO和屏幕阅读器依赖标题层级导航。无 <h1> 大纲不完整，多个则模糊文档结构。
语义 D5-4: <i> 和 <em> 的区别？
✅ <em> 表示语义强调（屏幕阅读器会用不同语调朗读），<i> 只是视觉斜体（术语/外文）。图标用 <i> 是历史习惯，语义上用 <span class="icon"> 更合理。
语义 D5-5: <article> 和 <section> 的误用？
✅ <article> 是独立可分发的内容单元，<section> 是主题分组。每个 <section> 应有标题。
语义 D5-6: 表单不用 <label> 的后果？
✅ 点击标签不能聚焦输入框、屏幕阅读器无法关联标签和输入。必填：每个 <input> 有对应 <label for="...">。
语义 D5-7: <nav> 里放非导航链接可以吗？
✅ <nav> 只用于主要导航区块。页脚的法律链接、社交媒体链接用量 <nav> 不合适。
语义 D5-8: <table> 用于页面布局有什么问题？
✅ 屏幕阅读器会按表格方式朗读（行/列/单元格），且响应式适配困难。布局用CSS（flex/grid）。
语义 D5-9: <button type="submit"> 在 <div> 中点击触发表单提交？
✅ 不在 <form> 内的 <button> 默认 type="submit"，点击会触发最近form的提交。应显式 type="button"。
语义 D5-10: <details> 和 <summary> 展开/折叠的内容对SEO有影响吗？
✅ 折叠内容仍然在DOM中，Google会索引，但 name 属性可以让一组details互斥展开。

🖼 D6 图像格式与优化

图像 D6-1: 装饰性背景图和内容图的正确处理方式区别？
✅ 装饰图用CSS background-image（不会被屏幕阅读器读），内容图用 <img> + alt。
图像 D6-2: 没有 srcset 在 Retina 屏上的问题？
✅ 2x/3x 屏上图片模糊。应用 srcset 提供多倍率版本，加 sizes 让浏览器选合适的。
图像 D6-3: WebP 格式不加 <picture> 降级的后果？
✅ 老Safari（14以下）不支持WebP，图片不显示。应用 <picture><source srcset="..." type="image/webp"><img src="fallback.jpg"></picture>。
图像 D6-4: loading="lazy" 在首屏图片上加的后果？
✅ 首屏图片应该立即加载，懒加载导致LCP延迟。只在非首屏图片上加 loading="lazy"。
图像 D6-5: SVG 中不含 viewBox 的后果？
✅ 没有 viewBox 的SVG无法正确缩放，设置宽高后可能被裁剪或溢出。
图像 D6-6: PNG 和 WebP/AVIF 在照片上的文件大小差距？
✅ 同样画质下WebP比PNG小约26%，AVIF再省约20%。照片用JPEG/WebP/AVIF，图标/截图用PNG或SVG。
图像 D6-7: 没有指定 width 和 height 的图片导致什么性能问题？
✅ 导致布局偏移（CLS）。浏览器预留空间，图片加载后页面不跳动。即使响应式也应设宽高比，配合 aspect-ratio。
图像 D6-8: object-fit: cover 和 object-position 不加的默认行为？
✅ 默认 object-fit: fill 会拉伸变形。且 object-position: 50% 50% 居中对齐可能裁掉重要内容（如人脸）。
图像 D6-9: 字体图标（Font Awesome）相对于SVG的劣势？
✅ 字体图标在低DPI下模糊、无法做多色、屏幕阅读器需要额外处理。SVG图标可缩放、可上色、可动画。
图像 D6-10: Base64 内嵌图片的适用场景和问题？
✅ 体积增大约33%、不缓存、阻塞渲染。仅适用于极小图标（< 1KB）且HTTP/1.1时代减少请求数。HTTP/2下应直接引用文件。

🌐 D7 浏览器兼容

兼容 D7-1: -webkit- 前缀什么时候还需要？
✅ 2025年大部分标准已稳定，但 backdrop-filter 在Safari上仍需 -webkit-backdrop-filter。
兼容 D7-2: position: sticky 在哪些情况下不生效？
✅ 父容器有 overflow: hidden/auto/scroll 时失效。且父容器高度必须大于sticky元素。
兼容 D7-3: CSS env(safe-area-inset-top) 什么时候用？
✅ iPhone刘海屏/圆角屏上，页面边缘被屏幕圆角遮挡。不处理则内容在刘海区域不可见。应加 padding-top: env(safe-area-inset-top)。
兼容 D7-4: 100vh 在移动端Safari上为什么不是全屏高？
✅ 移动端Safari的 100vh 包含底部工具栏，滚动时工具栏收起后多出一截空白。应用 dvh（动态视口）或 100%。
兼容 D7-5: gap 在Grid中哪些浏览器不支持？
✅ 现代浏览器全支持。但 gap 在Flex中的支持比Grid晚，Safari 14.1+才支持。
兼容 D7-6: accent-color 的作用和兼容性？
✅ 统一设置checkbox/radio/range的原生颜色，现代浏览器基本支持。不支持时也不影响功能。
兼容 D7-7: scroll-behavior: smooth 在哪些浏览器上不支持？
✅ 不支持时只是瞬间跳转，不影响功能。但不应依赖它做关键交互。
兼容 D7-8: <dialog> 元素的backdrop如何自定义样式？
✅ 用 ::backdrop 伪元素设背景色/模糊。老浏览器不支持 <dialog>，需polyfill或自定义弹窗。
兼容 D7-9: has() 选择器在哪些场景还不能依赖？
✅ Firefox 121+、Chrome 105+、Safari 15.4+ 已支持，基本可放心用。但不支持时整个选择器失效，不能做主要样式依赖。
兼容 D7-10: 不写 @supports 查询的新CSS特性对老浏览器的影响？
✅ 整个选择器/属性对不支持的浏览器直接忽略。安全的新特性用 @supports 包裹，同时提供降级方案。

🔒 D8 HTML安全

安全 D8-1: innerHTML 不加转义有什么风险？
✅ 用户输入直接拼接到 innerHTML 可执行任意脚本（XSS）。应用 textContent 或 sanitize-html。
安全 D8-2: <a href="javascript:..."> 为什么不安全？
✅ 点击触发JS执行，可被XSS利用。且SEO爬虫会跟链接。应用事件监听器处理交互。
安全 D8-3: <iframe> 不加 sandbox 的后果？
✅ 嵌入页面可执行脚本、提交表单、打开弹窗。应用 sandbox="allow-scripts allow-same-origin" 按需放权。
安全 D8-4: <form> 不加CSRF token的后果？
✅ 攻击者可伪造请求提交表单。应在表单中嵌入随机token，服务端验证。
安全 D8-5: target="_blank" 在链接上不加 rel="noopener" 的隐患？
✅ 新页面通过 window.opener 可以控制原页面（tab-napping攻击）。应始终加 rel="noopener noreferrer"。
安全 D8-6: onerror 在img标签上的潜在风险？
✅ 即使 onerror="this.onerror=null" 做了防循环，仍可被用于tracking。且 onerror 中可以执行任意JS。
安全 D8-7: CSP（Content-Security-Policy）头部不加的最常见后果？
✅ 页面可以加载任意第三方脚本，内联脚本不受限制，XSS防护弱。至少应设置 script-src 'self'。
安全 D8-8: localStorage 存储敏感信息（如token）的风险？
✅ 同源下任何JS都可读取，XSS攻击可直接窃取。敏感信息存 httpOnly cookie。
安全 D8-9: postMessage 不验证origin的漏洞？
✅ 任意网站可以向你的页面发消息，导致数据泄露。接收时始终验证 event.origin。
安全 D8-10: autocomplete="off" 在密码框上不怎么起作用？
✅ 现代浏览器忽略密码框的 autocomplete="off"，以保护密码管理功能。应用 autocomplete="new-password"。

✨ D9 动画与交互

动画 D9-1: 动画用 top/left 还是 transform？
✅ top/left 触发Layout→Paint→Composite，transform 仅触发Composite。GPU加速的 transform 流畅60帧。
动画 D9-2: opacity 动画中隐藏元素的正确处理？
✅ opacity: 0 的元素虽不可见但仍在DOM中接收点击事件。应同时加 pointer-events: none。
动画 D9-3: CSS动画结束时突然跳回起始状态的原因？
✅ animation-fill-mode: forwards 没加，动画结束后样式回到初始值。
动画 D9-4: transition: all ... 有什么坏处？
✅ 所有属性变化都触发过渡，包括不影响视觉的属性（如 color），增加GPU负载。应指定具体属性。
动画 D9-5: overflow: hidden 在动画元素上为什么可能导致卡顿？
✅ 强制浏览器创建新的裁剪区域，阻止GPU层合并。动画元素上慎用 overflow。
动画 D9-6: 为什么 display: none → block 不能过渡？
✅ display 不是可插值属性，没有中间状态。方案：用 visibility + opacity + position: absolute。
动画 D9-7: prefers-reduced-motion: reduce 时不处理微交互动效的后果？
✅ 用户系统设置减少动效时，闪烁/跳动/缩放动画可能引发前庭障碍患者眩晕。应关停或替换为无动效过渡。
动画 D9-8: @keyframes 中 100% 以外的状态没定义时浏览器怎么处理？
✅ 默认从元素原始样式开始到 100%，不是从 0% 开始。应显式定义 0% 和 100%。
动画 D9-9: 为什么 height: auto 不能过渡到固定高度？
✅ auto 不是可插值的关键帧值。方案：用 max-height 技巧或 grid-template-rows: 0fr → 1fr。
动画 D9-10: requestAnimationFrame 中修改 display 或 width 导致的问题？
✅ 在rAF中修改布局属性会强制同步布局（forced reflow），抵消rAF的性能优势。rAF中只改复合属性（transform/opacity）。

🔤 D10 字体与排版

字体 D10-1: 加载太多字重（如4种以上）的影响？
✅ 每个字重是一个文件，总下载量可达数MB。只加载用到的字重，或使用可变字体（Variable Fonts）。
字体 D10-2: 可变字体（Variable Fonts）的好处？
✅ 一个文件包含所有字重/字宽/倾斜，比独立字重文件小约40%。可用 font-weight: 350-700 精细控制。
字体 D10-3: 不声明 font-display 的默认行为？
✅ 默认 font-display: auto，浏览器策略因浏览器而异。Chrome约3秒不可见期（FOIT），之后用回退字体。
字体 D10-4: unicode-range 什么时候用？
✅ 英文和中文字体混合时，不设 unicode-range 会下载整个中文字体包（~5MB＋）。应用 unicode-range 只加载需要的字符集。
排版 D10-5: line-height 不带单位的优势？
✅ line-height: 1.5 相对于自身 font-size 计算，子元素继承的是因子不是绝对值。带单位（如 line-height: 24px）在大字号下会溢出。
排版 D10-6: 中英文混排时 word-break 和 line-break 的设置？
✅ 中文默认不换行，长URL会溢出。应用 word-break: break-word 或 overflow-wrap: break-word。
排版 D10-7: text-rendering: optimizeLegibility 在大段文字上的性能影响？
✅ 提升连字和字距但CPU开销大，在长文上可能导致滚动卡顿。只用于标题，正文用默认。
排版 D10-8: 中文网站用 letter-spacing: -0.5px 在哪些浏览器上模糊？
✅ 非整数像素的字间距在低DPI屏上渲染模糊。建议用 em 单位（如 letter-spacing: -0.02em）保持比例。
排版 D10-9: font-smooth / -webkit-font-smoothing 的跨浏览器问题？
✅ antialiased 在macOS上可能让文字变细，Windows上无效。最佳实践：只在macOS的深色背景上应用 -webkit-font-smoothing: antialiased。
排版 D10-10: 响应式排版中 px 比 rem 差在哪里？
✅ 用户更改浏览器默认字号（如老年人设为18px），px 固定不变不跟随。正文基础字号应设为 font-size: 100%（=16px），其他用 rem。
