⚡ 核心能力

🔍 智能检索 grep / glob

跨文件搜索代码结构、函数定义、字符串引用。支持正则与精确匹配，自动显示上下文。

// 搜索所有工具类定义
grep("class.*Plugin", "lib/")
// 查找文件
glob("**/*.test.js")

✏️ 精确编辑 edit_file

行模式 / 文本模式两种编辑方式。支持多文件并行编辑，合并后自动构建验证。

// 替换精确文本
edit_file({
  old_string: "旧代码",
  new_string: "新代码"
})

🌐 浏览器操控 Playwright

导航、截图、表单填写、点击、拖拽、网络请求捕获。支持无障碍快照交互。

browser_navigate browser_snapshot browser_click browser_type browser_screenshot browser_network

🌍 联网搜索 web_search

实时查询文档、API、库信息。搜索结果可直接抓取页面的 Markdown 内容。

web_search("tailwindcss v4 安装")
web_fetch("https://docs.rs/...")

▶ 典型工作流

🚀 Bug 修复 — 从报错到验证完成

① 复现错误
→
② 搜索根因
→
③ 编辑修复
→
④ 验证通过
→
⑤ Commit
Step 1
复现 — 运行报错命令，看到真实错误信息
Step 2
诊断 — grep 搜索相关代码，确定根因
Step 3
修复 — 编辑文件，注意全局一致性检查
Step 4
验证 — 自动循环：修复失败则回滚，最多 3 轮
Step 5
提交 — 里程碑自动 commit，保留验证记录
🎨 设计规范

🎯 输出风格

  • Bug 修复 → 温度 0.1，唯一方案
  • 代码生成 → 温度 0.2，直接实现
  • 架构设计 → 温度 0.4，2-3 方案对比
  • 头脑风暴 → 温度 0.8，多角度发散

📐 视觉标准

  • 无 SQL / shell 拼接
